# Hardening Batch 1 Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the highest-priority project risks by hardening backend request handling, correcting error semantics, preventing two frontend crash paths, and restoring a runnable backend test workflow.

**Architecture:** Keep the current product behavior intact, but extract the risky decisions into small pure helpers that can be covered with narrow tests. Harden the backend router with explicit validation and status-code handling, and move the frontend crash-prone logic behind typed helper functions so the fixes can be verified with `node:test` without introducing a browser test runner.

**Tech Stack:** FastAPI, httpx, React 19, TypeScript, Zustand, `node:test`, `unittest`, pytest via `uv`
